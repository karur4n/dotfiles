# open-with-zed

いまこの会話で作業しているプロジェクト／package のルートを zed で開く。

## 手順

1. **作業中のルートを判断する。** 起動時の cwd ではなく、この会話で実際に手を入れているディレクトリを文脈から決める。判断材料:
   - 直近で Edit / Write したファイル群の共通の親
   - 直前まで議論していた package / プロジェクトのルート（`package.json` や `.git` のあるディレクトリまで上る）
2. **迷ったら確認する。** 候補が複数ある、または判断がつかないときは推定せず、どのパスを開くかユーザーに聞く。
3. **開く。** 確定したパスに対して `zed <path>` を実行する。
